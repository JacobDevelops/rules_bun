# docs

## Purpose
Reference documentation for all `rules_bun` rules and extensions. `rules.md` is auto-generated from Starlark docstrings via Stardoc and must not be manually edited. `bun_install.md` and `index.md` are hand-authored.

## Key Files

| File | Description |
|------|-------------|
| `index.md` | Documentation site index — links to all sub-docs and provides a getting-started overview |
| `bun_install.md` | Full reference for the `bun_install` Bzlmod extension: all attributes, workspace patterns, isolation behavior |
| `rules.md` | **Auto-generated** rule attribute reference — regenerated by `bazel build //docs:rules_md` |
| `BUILD.bazel` | Stardoc target that generates `rules.md` from `//bun:defs_bzl` |

## For AI Agents

### Working In This Directory
- **Never edit `rules.md` by hand** — it is overwritten by Stardoc
- To refresh after changing rule attrs: `bazel build //docs:rules_md && cp bazel-bin/docs/rules.md docs/rules.md`
- `index.md` and `bun_install.md` are hand-maintained; edit these directly
- The docs site is published via `.github/workflows/pages.yml`

### Testing Requirements
```bash
bazel build //docs:rules_md  # verify Stardoc still produces output
```

## Dependencies

### Internal
- `//bun:defs_bzl` — Stardoc source for `rules.md` generation

### External
- `stardoc` 0.7.2 — Starlark documentation generator
